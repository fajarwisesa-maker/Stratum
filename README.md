# Stratum — AI-Powered Executive Dashboard for Product Quality Testing
**Case: SPL Biscuit Quality Testing · TÜV NORD Indonesia**

> **MVP Dataset Note:** This MVP currently uses the UCI Wine Quality dataset as a public proxy dataset to validate Stratum's ML and explainability pipeline. The architecture is designed to be retrained with actual SPL biscuit lab data when available.

---

## Architecture

```
Stratum1/
├── backend/                  # FastAPI + XGBoost ML backend
│   ├── api_main.py           # FastAPI app (all endpoints)
│   ├── pipeline.py           # ML training pipeline
│   ├── requirements.txt
│   ├── data/
│   │   ├── winequality-red.csv
│   │   └── winequality-white.csv
│   ├── artifacts/            # Generated by pipeline.py
│   │   ├── model.joblib
│   │   ├── metrics.json
│   │   ├── shap_beeswarm.png
│   │   ├── shap_waterfall_fail.png
│   │   ├── shap_waterfall_pass.png
│   │   └── shap_summary.json
│   └── utils/
│       ├── schema.py         # Pydantic request/response models
│       └── preprocessing.py  # Column normalization
└── src/                      # React/Vite frontend
    ├── api/client.js         # API client (fetch wrapper)
    ├── data/featureSchema.js # Wine Quality feature definitions
    └── views/                # Page components
```

## Quick Start

### 1. Backend setup

```bash
cd backend
pip install -r requirements.txt

# Train model + generate SHAP artifacts (run once)
python pipeline.py

# Start API server
uvicorn api_main:app --reload --port 8000
```

The API will be available at `http://localhost:8000`.  
Interactive docs: `http://localhost:8000/docs`

### 2. Frontend setup

```bash
# From Stratum1/
cp .env.example .env        # adjust VITE_API_BASE_URL if needed
npm install
npm run dev
```

Frontend runs at `http://localhost:5173`.

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/health` | Health check + model status |
| POST | `/predict` | Single sample prediction with SHAP |
| POST | `/batch` | CSV upload → bulk predictions |
| POST | `/simulate` | Modify one parameter, get new prediction |
| GET | `/metrics` | Model CV metrics |
| GET | `/shap-summary` | Feature importance JSON |
| GET | `/artifacts/shap_beeswarm.png` | SHAP beeswarm image |
| GET | `/artifacts/shap_waterfall_fail.png` | SHAP waterfall (FAIL) |
| GET | `/artifacts/shap_waterfall_pass.png` | SHAP waterfall (PASS) |

## Testing the API

### Health
```bash
curl http://localhost:8000/health
```

### Single Predict
```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{"parameters":{"fixed_acidity":7.4,"volatile_acidity":0.7,"citric_acid":0.0,"residual_sugar":1.9,"chlorides":0.076,"free_sulfur_dioxide":11,"total_sulfur_dioxide":34,"density":0.9978,"ph":3.51,"sulphates":0.56,"alcohol":9.4}}'
```

### Batch (CSV upload)
```bash
curl -X POST http://localhost:8000/batch \
  -F "file=@backend/data/winequality-red.csv"
```

### Simulate
```bash
curl -X POST http://localhost:8000/simulate \
  -H "Content-Type: application/json" \
  -d '{"parameters":{"fixed_acidity":7.4,"volatile_acidity":0.7,"citric_acid":0.0,"residual_sugar":1.9,"chlorides":0.076,"free_sulfur_dioxide":11,"total_sulfur_dioxide":34,"density":0.9978,"ph":3.51,"sulphates":0.56,"alcohol":9.4},"modified_param":"alcohol","new_value":12.5}'
```

---

## CSV Format

The upload accepts:
- Semicolon-separated (UCI format): `fixed acidity;volatile acidity;...`
- Comma-separated (snake_case): `fixed_acidity,volatile_acidity,...`

Column `quality` and `sample_id` are optional — they will be ignored if present.

## What is still mock / fallback

- **Demo Mode** (⚡ Load Demo Data button): Uses locally generated mock data — no API required.
- **SimulatorPanel** (Dashboard quick sim): Falls back to a local linear estimate if the API is unavailable, with a visible warning.
- **SHAP waterfall on Dashboard**: Shows top-5 SHAP values from the batch response (not the full 11-feature waterfall from the SHAP image).
- **Cluster Map**: Uses mock 2D cluster positions (PCA-like); a real PCA projection would require an additional `/cluster` endpoint.
